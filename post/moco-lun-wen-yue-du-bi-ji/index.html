MOCO 论文阅读笔记

2023-08-10 8 min read # ssl # cv

MOCO，全称Momentum Contrast for Unsupervised Visual Representation Learning[1]，是一种比较基础的自监督对比学习模型

代理任务

代理任务（pretext task）帮助模型学习数据中的有用特征和模式，以便在后续任务中进行更好的泛化。在BERT中，代理任务就是Masked Language Modeling、Next Sentence Prediction

Instance Discrimination

在论文[2]中，作者发现如果将一张豹子的图片喂给一个已经用有监督学习方式训练好的分类器，会发现他给出来的分类结果排名前几的全都是跟豹子相关的。他认为图片聚集在一起的原因并不是因为它们有相似的语义标签，而是因为这些照片长得太像了

因此在实例区分中，每一个instance都是一个类别，也就是每一张图片都看作是一个类别，训练目标是学习一种特征，能把每一个图片都区分开来

对一张图片XXX，通过数据增强等方法，得到X′X'X′，则这两者为正样本。数据集中的其他图片则为负样本

这里得到X′X'X′，还有许多方法：

  • 一张图片的不同view
  • 一张图片经过不同的encoder forward得到的向量

MOCO模型

Contrastive loss

对比学习希望模型学习有用的特征，要求模型在嵌入空间中将相似的样本靠近，而将不相似的样本分开。因此提出了Contrastive loss，目标是：

  • 相近的样本之间的距离越小越好
  • 相远的样本之间的距离越大越好

Memory Bank、Queue改进

对比学习能学习到一个更robust的特征表示的前提，是要有足够多的负样本进行训练。以ImageNet为例，如果数据集中的所有其他图片为负样本，就要存128万个图片。在做SGD优化时，假设mini-batch中有NNN个图片，经过Instance Discrimination过后，有2N−22N-22N−2个负样本。如果要把128万全部存入mini-batch，计算成本过大。
